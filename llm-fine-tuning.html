Project Goal

Security-Aware Incident Summarization

Fine-tune a base LLM on security incident reports and SOC tickets to generate concise, accurate summaries and recommended next steps for analysts.

The focus is on parameter-efficient fine-tuning (PEFT/LoRA) to keep costs manageable while preserving base model capabilities and minimizing catastrophic forgetting.

Tech Stack

Stack & Infrastructure

Python Hugging Face Transformers PEFT / LoRA Datasets PyTorch Docker FastAPI SageMaker / Vertex / Azure ML

Deployment: containerized inference endpoint with safety filters and logging for auditability.

Approach

PEFT/LoRA & Safety Considerations

  • Data Curation: Anonymized SOC tickets, incident reports, and playbooks with PII removed.
  • PEFT/LoRA: Attach low-rank adapters to attention layers instead of full fine-tuning.
  • Safety: Guardrails against leaking sensitive data, hallucinating non-existent incidents, or suggesting unsafe actions.
  • Catastrophic Forgetting: Mix general-domain data with security data; evaluate on both to ensure balanced performance.

Training Notebook Flow

  1. Load base model and tokenizer from Hugging Face Hub.
  2. Apply PEFT/LoRA configuration and prepare training dataset.
  3. Train with mixed precision and gradient accumulation.
  4. Evaluate on held-out incidents and generic tasks.
  5. Export adapter weights and package into Docker image.
Example

Sample Fine-Tuning Snippet (Pseudo-Code)

from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model
base_model = "meta-llama/Llama-3-8b"  # example
tokenizer = AutoTokenizer.from_pretrained(base_model)
model = AutoModelForCausalLM.from_pretrained(base_model)
config = LoraConfig(
    r=8,
    lora_alpha=16,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)
model = get_peft_model(model, config)
# Train on security incident summaries...
# Save adapter weights and push to private registry.
Evaluation

Metrics & Human Review

  • ROUGE/BLEU for summary quality vs. human-written baselines.

  • Red-team prompts to test for unsafe or speculative outputs.
