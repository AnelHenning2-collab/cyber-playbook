Multimodal Security Assistant

Fusing text, logs, and images/screenshots for richer incident response.

Project Goal

Vision-Language for Security Workflows

Explore a multimodal assistant that can reason over logs, text descriptions, and screenshots (e.g., phishing emails, admin consoles, SIEM dashboards) to support incident response.

Inspired by LLaVA-style vision-language models, this project focuses on input fusion challenges and how multimodal context can reduce ambiguity in security investigations.

Tech Stack

Stack & Models

Python Vision-Language Models (VLMs) LLaVA-style architectures CLIP / Image Encoders Transformers Diffusion Models (synthetic data) Streamlit / Gradio

Approach

Input Fusion & Synthetic Data

  • Input Fusion: Combine log snippets, textual descriptions, and screenshots into a unified representation.
  • Vision-Language: Use pre-trained VLMs to encode screenshots and fuse with text embeddings.
  • Diffusion Models: Generate synthetic security screenshots (e.g., fake admin panels, alerts) to augment scarce data.
  • Tasks: Explain what is happening in a screenshot, highlight suspicious UI elements, and correlate with logs.
Challenges

Nuances & Limitations

  • Aligning visual and textual context without overfitting to specific tools or vendors.
  • Ensuring screenshots are properly anonymized and free of sensitive data.
  • Latency and GPU requirements for real-time multimodal inference.
Future Work

Next Steps

  • Integrate with SOC triage assistant as a multimodal extension.
  • Evaluate with real analysts on usability and trust.
  • Explore lightweight on-device models for sensitive environments.
Reflection

What I Learned

Multimodal AI is promising for security, but data governance and privacy are even more critical when images and UI screenshots are involved. The main insight is that multimodal context can clarify ambiguous alerts, but only when carefully curated and aligned with human workflows.
