Project Goal

Robust Phishing & Malicious URL Detection

Build a production-ready phishing detection pipeline that combines classic ML (scikit-learn/XGBoost) with explainable features and careful handling of imbalanced data.

The model targets both email content and URLs, focusing on real-world datasets and evaluation that reflect noisy, adversarial environments.

Tech Stack

Stack & Datasets

Python Pandas Scikit-learn XGBoost SHAP Imbalanced-learn Docker Streamlit / Gradio

Datasets: SS Hacker Team Cybersecurity GLOBAL Challenge 2026, Malicious URL Detection (Enhanced 2026), Phishing and Legitimate Emails Dataset (2026).

Approach

Handling Imbalanced, Noisy Security Data

  • Feature Engineering: URL lexical features, domain age, WHOIS info, email header features, HTML structure signals.
  • Imbalanced Data: Class weights, SMOTE/SMOTE-NC, and threshold tuning for recall vs. precision trade-offs.
  • Modeling: Baseline logistic regression → tuned XGBoost/RandomForest with cross-validation.
  • Explainability: SHAP values to highlight which features drive phishing predictions for each sample.
Results

Metrics & Evaluation

  • ROC-AUC and PR-AUC on held-out test set.
  • Per-domain performance to detect overfitting to specific providers.
  • False positive rate at fixed recall (e.g., 95%) to reflect SOC tolerance.
  • Calibration curves to ensure probability outputs are trustworthy.
Responsible AI

Bias & Robustness

  • Check for bias against specific TLDs or regions.
  • Adversarial testing with obfuscated URLs and homoglyph attacks.
  • Explainability reports for security leadership and auditors.
Reflection

What I Learned & Future Work

Phishing detection is a classic but still evolving problem. The main lesson is that feature quality and evaluation design matter more than chasing the latest model. Handling imbalance and adversarial behavior is critical for real-world deployment.

Next steps: integrate this classifier into an email gateway or SOC pipeline, add online learning for evolving campaigns, and explore combining classic ML with lightweight transformer-based text encoders.
